How to use variables in your process

Click on the user task in your camunda modeler. Navigate to "Forms". Add a form field (at the moment only boolean is supported). Now add a property with the name "Q1" and put your question as a value.

E.g. add a variable "sunny". Now add the Property "Q1" with the value "Is it sunny outside?". Alexa will later ask the user that question to fill the value in the variable.

How to use the Generator

Requirements

  • VS Code with the Alexa Skills Toolkit Plugin installed
  • Alexa Skills Kit Developer Account
  • NPM
  • Git

Instructions

  1. Install Alexa Skills Kit (ASK) Toolkit, navigate to the extension and sign in with your developer account.
  2. Create a new skill using that extension:
    1. Choose a skill name.
    2. Set the default language.
    3. Choose custom skill.
    4. Choose alexa-hosted (You can also provide your own backend resource, but I did not test it yet).
    5. Choose the hosting runtime Alexa-Hosted (Node.js) (The generator may support other runtimes in the future).
    6. Choose a hosting region.
    7. Choose a local directory (where the skill will be saved on your computer).
    8. Click create, the creation may take some minutes.
  3. Open the Alexa Skill Generator in your browser.
  4. Upload your BPMN model.
  5. Choose a wake word.
  6. Enter your camunda rest endpoint url.
  7. Click create skill.
  8. A download will start, containing your skill files.
  9. Unzip the files.
  10. Copy all the files to your skill-folder on your computer (replace the existing files!).
  11. Open a terminal in the folder of your skill and navigate to the lambda folder.
  12. Run npm install. This will install some additional dependencies for your skill.
  13. Stage all changes, commit and push them. Alternatively, You can also use the deployment section of the Alexa Skills Toolkit Plugin.
  14. The deployment may take some time.
  15. Now open the Alexa Developer console in your browser, select your created skill and go to the test section. Enable testing for the development stage.
  16. Start interacting with your skill
